## Why It Exists

<div class="rmm-note-grid">
  <article>
    <h3>Repeatable operations</h3>
    <p>Common endpoint tasks are packaged as scripts that can be reviewed, reused, and deployed consistently.</p>
  </article>
  <article>
    <h3>Clear naming</h3>
    <p>Script prefixes and categories make it easier to find the right automation for the job.</p>
  </article>
  <article>
    <h3>Community friendly</h3>
    <p>Docs, issue guidance, and contribution paths are built around practical RMM usage.</p>
  </article>
</div>
