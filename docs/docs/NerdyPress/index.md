## What You Can Configure

<div class="nt-feature-grid">
  <article>
    <span>01</span>
    <h3>Identity</h3>
    <p>Set the site title, tagline, repository slug, header logo, site logo, and favicon.</p>
  </article>
  <article>
    <span>02</span>
    <h3>Visual Theme</h3>
    <p>Choose palette, style preset, font pairing, button treatment, and card styling.</p>
  </article>
  <article>
    <span>03</span>
    <h3>Navigation</h3>
    <p>Build navigation links, social icons, footer links, and footer copy in one flow.</p>
  </article>
  <article>
    <span>04</span>
    <h3>Content</h3>
    <p>Create documentation pages with slugs, descriptions, markdown content, and live preview.</p>
  </article>
  <article>
    <span>05</span>
    <h3>Homepage</h3>
    <p>Configure primary calls to action and feature cards for a useful first screen.</p>
  </article>
  <article>
    <span>06</span>
    <h3>Integrations</h3>
    <p>Add search, Mermaid, sitemap, edit links, metadata, PWA, Open Graph, robots, analytics, and deployment targets.</p>
  </article>
</div>

## Build Flow

<div class="nt-process-grid">
  <article>
    <span>Design</span>
    <p>Choose the identity, theme options, navigation, and supporting links that define the project.</p>
  </article>
  <article>
    <span>Preview</span>
    <p>Draft docs pages and review markdown content before generating the finished site files.</p>
  </article>
  <article>
    <span>Export</span>
    <p>Download a ready-to-run VitePress project that can be installed, customized, and deployed.</p>
  </article>
</div>

## Generated Project

<div class="nt-code-preview">

```text
my-vitepress-site/
├── .vitepress/
│   ├── config.mts
│   └── theme/
├── public/
├── index.md
├── guide.md
├── package.json
└── README.md
```
